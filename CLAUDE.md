# CLAUDE.md

## Project

Atlas is a Week 4 AI-powered web application capstone. It helps users compare a resume against a target job description and generate a career roadmap.

## Product Rules

- Keep the MVP focused on resume plus job description analysis.
- Do not add auth, payments, LinkedIn scraping, or job-board integrations in v1.
- Do not store user resumes or job descriptions by default.
- Do not expose API keys client-side.
- Use clear validation errors for empty or invalid inputs.
- Treat fit score as guidance, not a hiring prediction.

## Engineering Rules

- Prefer small, testable modules.
- Keep prompt construction separate from UI code.
- Validate AI responses before rendering.
- Add tests for validation, prompt construction, and output parsing.
- Keep commits meaningful and scoped.

## Brand Rules

- Use the Atlas name and tagline: "Map your next career move."
- Use a clean, professional UI with restrained blue accents.
- Avoid copy that overpromises job outcomes.
